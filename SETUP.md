# 🛠️ Setup Guide — Medium Agent Bot

This guide walks you through every step to get the Medium Agent Telegram bot running on your local machine.

> [!NOTE]
> This project runs **entirely on your PC** — no cloud AI APIs. Article summaries are generated by [Ollama](https://ollama.com) running a local LLM. Paywalled articles are accessed via [Freedium](https://freedium.cfd).

## Step 1 — Install Python

You need **Python 3.10+** installed on your machine.

### Windows

1. Download from [python.org/downloads](https://www.python.org/downloads/)
2. During install, **check** ✅ "Add Python to PATH"
3. Verify:
   ```powershell
   python --version
   ```

### macOS / Linux

```bash
# macOS (via Homebrew)
brew install python@3.12

# Ubuntu/Debian
sudo apt update && sudo apt install python3 python3-pip python3-venv
```

---

## Step 2 — Install Ollama & Pull a Model

Ollama is the local LLM engine that powers article summarization.

### 2a. Install Ollama

| Platform | Method |
|----------|--------|
| **Windows** | Download from [ollama.com/download](https://ollama.com/download) → run the installer |
| **macOS** | Download from [ollama.com/download](https://ollama.com/download) or `brew install ollama` |
| **Linux** | `curl -fsSL https://ollama.com/install.sh \| sh` |

### 2b. Pull the Summarization Model

After installing, open a terminal and pull the default model:

```bash
ollama pull phi3.5
```

> [!TIP]
> **Model recommendations by hardware:**
>
> | Model | Size | RAM Needed | Best For |
> |-------|------|:----------:|----------|
> | `phi3.5` | 2 GB | 4 GB | ⭐ Default — fast, great at JSON output |
> | `llama3.2` | 2 GB | 4 GB | Good all-around quality |
> | `mistral` | 4 GB | 8 GB | Better reasoning, slightly slower |
> | `gemma2` | 5 GB | 8 GB | Great at structured output |
> | `qwen2.5` | 4 GB | 8 GB | Excellent multilingual support |

### 2c. Verify Ollama is Working

```bash
ollama run phi3.5 "Say hello in one sentence"
```

If you get a response, Ollama is ready. Press `Ctrl+D` or type `/bye` to exit.

> [!IMPORTANT]
> Ollama must be **running** whenever you use the bot. On Windows/macOS, the Ollama app runs in the system tray after installation. On Linux, start it with `ollama serve`.

## Step 3 — Clone & Install Dependencies

```bash
# Clone the repository
git clone https://github.com/your-username/mediumAgent.git
cd mediumAgent

# Create a virtual environment (recommended)
python -m venv venv

# Activate it
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate

# Install Python packages
pip install -r requirements.txt

# Install browser for scraping (one-time)
playwright install chromium
```

> [!NOTE]
> The `playwright install chromium` command downloads a Chromium browser (~140 MB) that the scraper uses in headless mode. This is a one-time download.

## Step 4 — Log in to Medium

Medium uses passwordless login (Google, Apple, magic link, etc.), so we log in **once** via a real browser and save the session.

```bash
python login.py
```

This opens a **visible browser window**:

1. Log in to Medium using any method (Google, email link, etc.)
2. Once you see your Medium home feed, switch back to the terminal
3. Press **ENTER** to save the session

Your session is saved to `session.json` and reused automatically by the bot. Re-run `python login.py` if it ever expires (typically lasts weeks/months).

> [!NOTE]
> No passwords are stored anywhere — the bot only uses browser cookies from your saved session.

## Step 5 — Configure `.env`

### 5a. Create your `.env` file

```bash
# Windows
copy env.example .env

# macOS / Linux
cp env.example .env
```

### 5b. Fill in the values

Open `.env` in any text editor and configure each section:

---

#### 🤖 Ollama (Local LLM)

These usually don't need to be changed unless you picked a different model:

```env
OLLAMA_MODEL=phi3.5
OLLAMA_BASE_URL=http://localhost:11434
```

---

#### 🔓 Freedium (Paywall Bypass)

The bot uses Freedium to read members-only Medium articles. Default works out of the box:

```env
FREEDIUM_BASE_URL=https://freedium.cfd
```

> [!NOTE]
> If Freedium changes its domain in the future, just update this URL. No code changes needed.

---

#### 📧 Gmail App Password

To send email digests via the `/mail` command:

1. Go to [myaccount.google.com](https://myaccount.google.com) → **Security**
2. Turn on **2-Step Verification** (required)
3. Search for **App passwords** → select **Mail** → **Generate**
4. Copy the 16-character code

```env
GMAIL_USER=you@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
GMAIL_TO=you@gmail.com
```

> [!WARNING]
> Never commit your `.env` file to Git. The `.gitignore` should already exclude it.

---

#### 📬 Telegram Bot

1. Open Telegram → search **@BotFather** → send `/newbot`
2. Follow prompts → you'll receive a **bot token**
3. Start a chat with your new bot (send it any message like "hello")
4. Visit in your browser:
   ```
   https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
   ```
5. Find `"chat": {"id": 123456789}` → that number is your chat ID

```env
TELEGRAM_BOT_TOKEN=123456789:ABCdef...
TELEGRAM_CHAT_ID=987654321
```

> [!TIP]
> **Register bot commands with BotFather** for a better UX:
> 1. Message @BotFather → send `/setcommands`
> 2. Select your bot
> 3. Send:
>    ```
>    start - Show welcome message and usage
>    mail - Scan feed and send email digest
>    ```
> This adds a command menu to your bot's chat.

## Step 6 — Run the Bot

Make sure Ollama is running (check system tray or run `ollama serve`), then:

```bash
python bot.py
```

You should see:

```
00:00:00  INFO      =======================================================
00:00:00  INFO      Medium Agent Bot — starting
00:00:00  INFO      =======================================================
00:00:00  INFO      Bot is running. Listening for chat ID: 987654321
00:00:00  INFO      Press Ctrl+C to stop.
```

Now go to Telegram and test:

1. Send `/start` → you should get the welcome message
2. Send any Medium article URL → you should get a summary back
3. Send `/mail` → you should receive a digest email

## Step 7 — Keep It Running

The bot needs to stay running to listen for messages. Here are options:

### Option A: Terminal (simple)

Just leave the terminal open with `python bot.py` running. On Windows, minimize the terminal.

### Option B: Background Process (Windows)

Create a batch file `start_bot.bat`:

```batch
@echo off
cd /d D:\Projects\mediumAgent
call venv\Scripts\activate
python bot.py
```

Then:
1. Press `Win + R` → type `shell:startup` → Enter
2. Place a shortcut to `start_bot.bat` in the Startup folder
3. The bot will start automatically when you log in

### Option C: Task Scheduler (Windows, hidden)

1. Open **Task Scheduler** → **Create Task**
2. **General** tab: Name it `Medium Agent Bot`, check "Run whether user is logged on or not"
3. **Trigger** tab: At startup
4. **Action** tab:
   - Program: `D:\Projects\mediumAgent\venv\Scripts\pythonw.exe`
   - Arguments: `bot.py`
   - Start in: `D:\Projects\mediumAgent`
5. Click OK

### Option D: systemd (Linux)

Create `/etc/systemd/system/medium-agent.service`:

```ini
[Unit]
Description=Medium Agent Telegram Bot
After=network.target

[Service]
Type=simple
User=your_username
WorkingDirectory=/path/to/mediumAgent
ExecStart=/path/to/mediumAgent/venv/bin/python bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Then:

```bash
sudo systemctl enable medium-agent
sudo systemctl start medium-agent
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `ConnectionError: Ollama is not running` | Start Ollama: `ollama serve` (or open the Ollama app) |
| `No saved session found` | Run `python login.py` to log in to Medium |
| `Medium session expired` | Run `python login.py` again to refresh your session |
| `Gmail send failed` | Ensure 2-Step Verification is ON and app password is correct |
| `Telegram error: chat not found` | Send a message to your bot first, then get the correct `chat_id` |
| `Model not found` | Run `ollama pull phi3.5` (or your chosen model) |
| Playwright hangs | Try `playwright install chromium` again |
| Bot doesn't respond | Check that `TELEGRAM_CHAT_ID` matches your actual chat ID |
| Freedium not loading | The domain may have changed — check [freedium.cfd](https://freedium.cfd) and update `FREEDIUM_BASE_URL` in `.env` |
